# Hypothesis Testing: Software vs Science

> [!note] Key Insight
> Software hypothesis testing is FASTER than traditional science because deployment is instant and rollback is free (git). Traditional science needs weeks for experiments; software scientists can test 7 hypotheses in 6 hours. But the rigor requirements are identical — sample size, statistical significance, and baseline comparison apply equally.

## Key Differences
| Aspect | Traditional Science | Software Science |
|--------|-------------------|-----------------|
| Experiment time | Days to months | Minutes to hours |
| Rollback cost | High (can't un-do) | Free (git revert) |
| Sample size | Fixed by experiment | Scalable by data |
| Peer review | Months | Adversarial validation (minutes) |
| Publication | Papers | Vault notes + npm deploy |

## What's the Same
- Hypotheses need mechanisms (WHY, not just WHAT)
- Falsification criteria are mandatory
- Statistical significance matters for data claims
- Base rate comparison prevents false positives
